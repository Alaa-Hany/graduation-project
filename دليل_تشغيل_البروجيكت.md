# 🚀 دليل تشغيل البروجيكت — Kinder World

---

## 📁 هيكل البروجيكت

```
Graduation Project/
├── kinderbackend/          ← Python FastAPI (الـ Backend)
│   ├── main.py             ← نقطة البداية
│   ├── models.py           ← جداول الـ Database
│   ├── routers/            ← الـ API endpoints
│   ├── core/               ← الإعدادات والـ middleware
│   ├── alembic/            ← migrations الـ database
│   ├── kinder.db           ← ملف الـ SQLite (الـ database الافتراضي)
│   ├── .env                ← متغيرات البيئة (مهم جداً)
│   └── .venv/              ← البيئة الافتراضية Python
│
└── kinder_world_child_mode/ ← Flutter App (الـ Frontend)
    ├── lib/                ← كود الـ Flutter
    ├── android/            ← ملفات Android
    └── pubspec.yaml        ← dependencies
```

## 1️⃣ تشغيل الـ Backend (Python FastAPI)

### الطريقة السهلة — بـ double-click على ملف الـ bat

افتح مجلد `kinderbackend` وشغّل:
```
start_server.bat
```

### الطريقة اليدوية — من Terminal

```powershell
# 1. روح على مجلد الـ backend
cd "C:\Graduation Project\kinderbackend"

# 2. شغّل الـ virtual environment
.venv\Scripts\activate

# 3. شغّل السيرفر
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

> `--reload` يعني السيرفر بيعيد نفسه تلقائياً لما تعدّل في الكود.

### ✅ علامة إن السيرفر شغّال

في الـ Terminal هتشوف:
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

## 2️⃣ الـ Database (SQLite)

### إيه هو الـ Database؟

الـ Database الافتراضي هو **SQLite** — يعني ملف واحد اسمه `kinder.db` في مجلد `kinderbackend`.
مفيش برنامج خارجي محتاج تشغّله، بيشتغل تلقائياً مع الـ backend.

### تشغيل الـ Migrations (لو أول مرة أو في تعديلات جديدة)

```powershell
cd "C:\Graduation Project\kinderbackend"
.venv\Scripts\activate

# تطبيق أحدث تعديلات الـ database
alembic upgrade head
```

### عرض الـ Database (بدون كود)

استخدم برنامج **DB Browser for SQLite** (موجود على جهازك):
1. افتح البرنامج
2. اضغط **Open Database**
3. اختر الملف: `C:\Graduation Project\kinderbackend\kinder.db`
4. هتشوف كل الجداول والبيانات

## 4️⃣ تشغيل Flutter App (الـ Frontend)

### متطلبات

- Flutter SDK مثبّت على جهازك
- محاكي Android (Android Emulator) أو موبايل متوصل بالـ USB

### الأوامر

```powershell
# 1. روح على مجلد الـ Flutter
cd "C:\Graduation Project\kinder_world_child_mode"

# 2. جيب الـ dependencies (أول مرة فقط أو لو في packages جديدة)
flutter pub get

# 3. شوف الأجهزة المتاحة
flutter devices

# 4. شغّل الأب على محاكي أو موبايل
flutter run

# أو لو عايز تختار جهاز معين
flutter run -d <device_id>
```

### تشغيل على المتصفح (أسهل للاختبار)

```powershell
flutter run -d chrome
```

## 5️⃣ الـ .env — متغيرات البيئة

ملف `.env` في `kinderbackend/` هو الإعدادات السرية. الموجود فيه حالياً:

```env
KINDER_JWT_SECRET=replace-with-strong-random-32-plus-chars-key-1
ENABLE_ADMIN_SEED_ENDPOINT=true
ADMIN_SEED_PASSWORD=Admin@123456
ADMIN_SEED_EMAIL=admin@kinderworld.app
```

> **لو DATABASE_URL فاضية** → بيستخدم SQLite تلقائياً (ملف `kinder.db`).

## 7️⃣ الأوامر الأكتر استخداماً — سريع

| المهمة | الأمر |
|--------|-------|
| تشغيل الـ backend | `cd kinderbackend && .venv\Scripts\activate && uvicorn main:app --reload` |
| فتح الـ API docs | `http://127.0.0.1:8000/docs` |
| تحديث الـ database | `alembic upgrade head` |
| تشغيل Flutter | `flutter run` |
| تشغيل الـ tests | `pytest -v` (backend) أو `flutter test` (flutter) |
| إنشاء migration جديد | `alembic revision --autogenerate -m "وصف التعديل"` |

## 8️⃣ تشغيل الكل بـ Docker (اختياري)

لو عندك Docker مثبّت، تقدر تشغّل كل حاجة بأمر واحد:

```powershell
cd "C:\Graduation Project\kinderbackend"
docker-compose up --build
```

ده بيشغّل:
- PostgreSQL database على port 5432
- Redis cache على port 6379
- Backend API على port 8000

لإيقافهم:
```powershell
docker-compose down
```

## ❗ مشاكل شائعة وحلها

| المشكلة | الحل |
|---------|------|
| `JWT secret not set` | تأكد إن `.env` موجود وفيه `KINDER_JWT_SECRET` |
| `Module not found` | شغّل `.venv\Scripts\activate` الأول |
| `Port 8000 in use` | غيّره: `uvicorn main:app --port 8001` |
| `flutter: SDK not found` | تأكد إن Flutter في الـ PATH |
| الـ database فاضية | شغّل `alembic upgrade head` |
